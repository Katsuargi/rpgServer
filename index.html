Text based Web RPG

In high school I played quite a few different text based browser games including RPGs. I thought this would be a good initial project because I'd be able to find a place to use just about anything I learned in JavaScript.

The baseline project is now finished. It features a character creation and display, a shop and shopping cart feature, a navigation process, a basic quest system, a basic leveling system, and random encounter / combat mechanic. Everything is streamlined well enough that adding new content(areas, items, monsters, quests, etc.) is the work of minutes.

There is room for improvement here with skills, classes, pictures, etc.

Navigation updated to a more general NSEW using coordinates. Other systems such as save / load updated to account for this.

Equipment system added and fixed. Added health potions. Added proper leveling system.

Data Crunching

Data crunch for a forum game I play in. Ship design in game features parts with variable values, and different categories of parts that impact the same stats. Involves complicated trade offs for cost/effect. Tool designed to eventually crunch all legal combinations of parts within a certain range using heuristics to keep things sane. Has progressed from linear single part cruncher up to the current general 2x2 configuration. Can easily be rote worked for any number of categories + parts from here. Lacks anything beyond basic heuristics, however, so lacks more than basic usefulness.

Rough estimates give about 10 parts per category, and 8 categories per major stat. Current heuristics include limiting any individual part to 12 (parts have diminishing returns. This is a basic sanity check as manual testing has shown even outlier cases don't want more than 12 of a given part). (12^8)*(10^8) is still a really, really big number. To say nothing of heuristics to actually track down the values with trade offs while weeding out combinations that are just flat inferior.
